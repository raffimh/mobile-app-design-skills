# Quality Gates Checklist

## P0 (Must Pass)
- [ ] All requirements are mapped to UI
- [ ] Every screen has states: success/loading/empty/error
- [ ] No dead-end flow
- [ ] Touch targets meet platform baseline
- [ ] Contrast & focus meet accessibility baseline
- [ ] Safe area and keyboard overlap risks are resolved
- [ ] Platform conventions match selected platform

## P1 (Should Pass)
- [ ] Tokens are consistent across screens
- [ ] High design system component reuse
- [ ] Microcopy is clear and domain-specific
- [ ] Error messaging is actionable
- [ ] Offline/network-degraded behavior is defined where relevant
- [ ] Performance risks are documented

## P2 (Nice to Have)
- [ ] Micro-interactions are meaningful and not excessive
- [ ] Visual rhythm is clean (spacing/alignment/balance)
- [ ] Additional edge cases are handled

## AI Slop Test (Must Pass)
- [ ] Not a domain-agnostic template
- [ ] Differentiation is structural/interactional, not only color styling
- [ ] Domain anchors are visible on each core screen

## Scoring Rubric (0-100)
- Design system consistency: 25
- UX/usability & flow clarity: 25
- Engineering feasibility: 25
- Visual polish & non-genericity: 25
